# 알려진 호스트

알려진 호스트는 서버의 신원이 변경되었는지 감지하기 위해 서버 지문을 저장합니다 (잠재적 보안 문제).

## 알려진 호스트 인터페이스

![Known Hosts Interface](/knownhosts.png)

## 알려진 호스트란?

SSH 서버에 처음 연결할 때 서버는 지문을 제시합니다. Marix는 이를 저장하여:
- 향후 연결 시 서버 신원 확인
- 중간자 공격 감지
- 서버 키가 예기치 않게 변경되면 경고

## 알려진 호스트 열기

사이드바에서 **"알려진 호스트"**를 클릭하세요.

## 알려진 호스트 보기

목록에 표시되는 내용:

| 열 | 설명 |
|--------|-------------|
| 호스트 | 서버 호스트명/IP |
| 포트 | SSH 포트 |
| 키 유형 | SSH 키 알고리즘 (ED25519, RSA, ECDSA) |
| 지문 | 고유한 서버 식별자 |
| 추가됨 | 호스트를 신뢰한 날짜 |

## 알려진 호스트 검색

1. 상단의 검색 상자 사용
2. 호스트명, IP 또는 지문으로 검색

## 키 유형별 필터

1. 드롭다운 필터 사용
2. 선택: 전체, ED25519, RSA 또는 ECDSA

## 서버에서 호스트 가져오기

연결하지 않고 서버의 지문을 수동으로 추가:

1. **"호스트에서 가져오기"** 버튼 클릭
2. 입력:
   - 호스트명 또는 IP
   - 포트 (기본값 22)
3. **"가져오기"** 클릭
4. 서버의 지문이 가져와져 저장됨

## 알려진 호스트 제거

1. 목록에서 호스트 찾기
2. **삭제** 버튼 (휴지통 아이콘) 클릭
3. 제거 확인

### 제거해야 할 때

- 서버가 재설치되어 새 키가 있는 경우
- 서버 IP/호스트명이 변경된 경우
- 더 이상 이 서버에 연결하지 않는 경우

## 호스트 키 확인

새 서버에 연결할 때:
1. Marix가 서버의 지문을 가져옴
2. 확인하고 신뢰하라는 요청을 받음
3. 수락하면 알려진 호스트에 저장됨
4. 향후 연결은 자동으로 확인됨

## 보안 경고

### "호스트 키가 변경되었습니다"
이 경고는 서버의 지문이 저장된 것과 다를 때 나타납니다.

**가능한 원인:**
- 서버가 재설치됨
- 서버의 SSH 키가 재생성됨
- ⚠️ **중간자 공격** (드물지만 가능)

**해결 방법:**
1. 서버 관리자에게 키가 정당하게 변경되었는지 확인
2. 확인되면 알려진 호스트에서 이전 항목 제거
3. 다시 연결하고 새 지문 수락
4. 의심스러우면 연결하지 않기

## 모범 사례

1. **새 서버에 연결할 때 지문 확인**
2. **변경된 호스트 키에 대한 경고 무시하지 않기**
3. **알려진 호스트 목록 정기적으로 검토**
4. **더 이상 사용하지 않는 서버의 오래된 항목 제거**
