# 백업 및 복원

암호화된 백업으로 데이터를 보호하세요. Marix는 Argon2id 암호화를 사용하는 6개의 백업 제공업체를 지원합니다.

> 💡 **팁:** 백업을 위해 클라우드 계정을 연결하면 동일한 장치에서 복원을 위해 다시 연결할 필요가 없습니다. 인증 정보가 로컬에 저장됩니다.

## 백업 대상

- 모든 서버 및 자격 증명 (비밀번호, 개인 키)
- 키체인의 SSH 키
- 포트 포워딩 구성
- 2FA/TOTP 비밀키
- 애플리케이션 설정
- Cloudflare API 토큰

## 비밀번호 요구 사항

| 요구 사항 | 설명 |
|-------------|-------------|
| 길이 | 최소 10자 |
| 대문자 | 대문자 포함 |
| 소문자 | 소문자 포함 |
| 숫자 | 최소 1개의 숫자 포함 |
| 특수문자 | 특수문자 포함 |

> ⚠️ **경고:** 비밀번호를 복구할 수 없습니다. 잊어버리면 백업에 영구적으로 접근할 수 없습니다.

## 로컬 백업

암호화된 백업 파일을 컴퓨터에 저장합니다.

### 로컬 백업 생성

1. **설정** → **"백업 생성"** 클릭
2. **로컬** 탭 (폴더 아이콘) 선택
3. **강력한 비밀번호** 입력 (10자 이상)
4. 비밀번호 확인
5. **"백업 생성"** 클릭
6. `.marix` 파일 저장 위치 선택
7. 이 파일을 안전하게 보관

### 로컬 백업에서 복원

1. **설정** → **"백업 복원"** 클릭
2. **로컬** 탭 선택
3. **"파일 선택"**을 클릭하고 `.marix` 백업 파일 선택
4. 백업 생성 시 사용한 **비밀번호** 입력
5. **"복원"** 클릭
6. 애플리케이션이 모든 데이터가 복원된 상태로 재시작

## Google Drive

백업을 Google Drive에 동기화합니다.

### 연결

1. **Google Drive** 탭 선택
2. **"연결"** 클릭
3. Google 계정으로 로그인
4. Marix에 권한 부여
5. 이메일과 함께 "연결됨" 표시

### 백업 생성

1. 연결 후 강력한 비밀번호 입력
2. 비밀번호 확인
3. **"백업 생성"** 클릭
4. 상태 표시: "백업이 성공적으로 생성되었습니다"

### 복원

1. 이미 연결되어 있으면 계정이 기억됨
2. 백업이 있으면 "마지막 백업: [날짜]" 표시
3. 비밀번호 입력
4. **"복원"** 클릭

## GitHub

개인 GitHub Gist에 백업을 저장합니다.

### 연결

1. **GitHub** 탭 선택
2. **"연결"** 클릭
3. **장치 코드**가 나타남 (예: `ABCD-1234`)
4. https://github.com/login/device로 이동
5. 코드 입력
6. Marix 승인

### 백업 생성

1. 강력한 비밀번호 입력
2. 비밀번호 확인
3. **"백업 생성"** 클릭
4. 백업이 개인 Gist로 저장됨

### 복원

1. 이미 연결되어 있으면 계정이 기억됨
2. 백업이 있으면 "마지막 백업: [날짜]" 표시
3. 비밀번호 입력
4. **"복원"** 클릭

### 내 백업은 어디에?

https://gist.github.com → 개인 Gist → "marix-backup" 찾기

## 제공업체 비교

| 제공업체 | 저장 위치 | 장점 | 단점 |
|----------|------------------|------|------|
| **로컬** | 내 컴퓨터 | 완전한 제어, 오프라인 | 파일 수동 관리 필요 |
| **Google Drive** | Google 클라우드 | 자동 동기화, 널리 사용됨 | Google 계정 필요 |
| **OneDrive** | Microsoft 클라우드 | Windows와 통합 | Microsoft 계정 필요 |
| **GitHub** | GitHub Gist | 무료, 버전 관리 | GitHub 계정 필요 |
| **GitLab** | GitLab Snippet | 무료, 자체 호스팅 옵션 | GitLab 계정 필요 |
| **Box.net** | Box.net 클라우드 | 기업 친화적 | Box 계정 필요 |

## 모범 사례

1. **다른 비밀번호 사용** - 제공업체마다 다른 비밀번호
2. **비밀번호 저장** - 비밀번호 관리자에 저장
3. **정기적으로 복원 테스트** - 백업이 작동하는지 확인
4. **로컬 백업 유지** - 보조 옵션으로
5. **백업 파일 공유 금지** - 민감한 데이터 포함
6. **정기적으로 백업 업데이트** - 새 서버 추가 후
