# 포트 포워딩 (SSH 터널)

SSH 터널은 SSH 연결을 통해 트래픽을 안전하게 라우팅합니다.

## 포트 포워딩 인터페이스

![Port Forwarding Interface](/portfoward.png)

## 포트 포워딩 유형

| 유형 | 방향 | 사용 사례 |
|------|-----------|----------|
| **로컬** | 원격 → 로컬 | 로컬 포트를 통해 원격 서비스 접근 |
| **원격** | 로컬 → 원격 | 원격 네트워크에 로컬 서비스 노출 |
| **동적** | SOCKS 프록시 | SSH를 통한 SOCKS 프록시 생성 |

## 포트 포워딩 열기

사이드바에서 **"포트 포워딩"**을 클릭하세요.

## 포트 포워드 생성

### 1단계: 대화 상자 열기
**"포워드 추가"** 버튼 클릭

### 2단계: 설정

| 필드 | 설명 | 예시 |
|-------|-------------|---------|
| 이름 | 설명적인 이름 | "MySQL to localhost" |
| 유형 | 로컬, 원격 또는 동적 | 로컬 |
| 서버 | 터널링할 SSH 서버 | My VPS |
| 로컬 호스트 | 로컬 바인드 주소 | 127.0.0.1 |
| 로컬 포트 | 로컬 포트 번호 | 3307 |
| 원격 호스트 | 원격 대상 | 127.0.0.1 |
| 원격 포트 | 원격 포트 | 3306 |

### 3단계: 저장
**"추가"** 또는 **"저장"** 클릭

## 사용 사례 예시

### 로컬 포워드: 원격 MySQL 접근

로컬 컴퓨터에서 서버의 MySQL 데이터베이스에 접근:

| 설정 | 값 |
|---------|-------|
| 유형 | 로컬 |
| 로컬 | 127.0.0.1:3307 |
| 원격 | 127.0.0.1:3306 |

**사용법:** `localhost:3307`에 연결하여 서버의 MySQL에 접근.

### 원격 포워드: 로컬 앱 노출

서버가 로컬 개발 서버에 접근할 수 있게 함:

| 설정 | 값 |
|---------|-------|
| 유형 | 원격 |
| 로컬 | 127.0.0.1:3000 |
| 원격 | 0.0.0.0:8080 |

**사용법:** 서버의 포트 8080이 로컬 localhost:3000에 연결.

### 동적: SOCKS 프록시

서버를 통한 브라우징을 위한 SOCKS5 프록시 생성:

| 설정 | 값 |
|---------|-------|
| 유형 | 동적 |
| 로컬 | 127.0.0.1:1080 |

**사용법:** `localhost:1080`에서 SOCKS5 프록시를 사용하도록 앱 설정.

## 포트 포워드 관리

### 시작
1. 목록에서 포워드 찾기
2. **시작** 버튼 (▶) 클릭
3. 상태가 "연결됨"으로 변경

### 중지
1. 실행 중인 포워드 찾기
2. **중지** 버튼 (■) 클릭
3. 상태가 "연결 끊김"으로 변경

### 편집
1. 실행 중이면 포워드 **중지**
2. **편집** 버튼 (연필) 클릭
3. 설정 수정
4. **저장** 클릭

### 삭제
1. 실행 중이면 포워드 중지
2. **삭제** 버튼 (휴지통) 클릭
3. 삭제 확인

## 상태 모니터링

목록에 표시되는 내용:
- **상태 표시기**: 녹색 = 연결됨, 빨간색 = 오류
- **연결 수**: 활성 연결
- **전송된 바이트**: 입/출력 트래픽

## 일반적인 사용 사례

| 서비스 | 로컬 포트 | 원격 포트 | 유형 |
|---------|------------|-------------|------|
| MySQL | 3307 | 3306 | 로컬 |
| PostgreSQL | 5433 | 5432 | 로컬 |
| Redis | 6380 | 6379 | 로컬 |
| MongoDB | 27018 | 27017 | 로컬 |
| Web App | 8080 | 80 | 로컬 |
| Jupyter | 8889 | 8888 | 로컬 |

## 문제 해결

### 포트가 이미 사용 중
- 다른 로컬 포트 선택
- 다른 애플리케이션이 해당 포트를 사용 중인지 확인
- 충돌하는 서비스 중지

### 연결 거부됨
- 원격 서비스가 실행 중인지 확인
- 서버의 방화벽 확인
- 올바른 원격 호스트/포트인지 확인

### 터널 끊김
- SSH 연결 안정성 확인
- 서버에 시간 초과 설정이 있을 수 있음
- keep-alive 사용 고려
