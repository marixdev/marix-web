# SSH 키 관리자

서버에 대한 비밀번호 없는 인증을 위해 SSH 키를 관리합니다.

## SSH 키 관리자 인터페이스

![SSH Key Manager Interface](/sshkey.png)

## SSH 키 관리자 열기

사이드바 메뉴에서 **"SSH 키"**를 클릭하세요.

## 새 SSH 키 생성

### 1단계: 생성기 열기
**"키 생성"** 버튼 클릭

### 2단계: 세부 정보 입력

| 필드 | 설명 | 필수 |
|-------|-------------|----------|
| 키 이름 | 이 키를 식별하는 이름 (예: "work-laptop") | 예 |
| 키 유형 | ed25519 (권장), RSA 또는 ECDSA | 예 |
| 암호구 | 키를 암호화하는 비밀번호 (선택 사항이지만 권장) | 아니오 |
| 설명 | 추가 메모 (예: 이메일 주소) | 아니오 |

### 3단계: 생성
**"생성"** 클릭

### 4단계: 완료
키가 목록에 나타나며 다음이 표시됩니다:
- 이름
- 유형 배지 (ED25519, RSA, ECDSA)
- 지문
- 생성 날짜

## 키 유형 비교

| 유형 | 보안 | 호환성 | 권장 |
|------|----------|---------------|----------------|
| **ED25519** | 우수 | 최신 서버 | ✅ 권장 |
| **RSA (4096)** | 매우 좋음 | 범용 | 이전 시스템에 좋음 |
| **ECDSA** | 좋음 | 대부분의 서버 | ED25519 대안 |

## 기존 키 가져오기

### 1단계: 가져오기 도구 열기
**"키 가져오기"** 버튼 클릭

### 2단계: 키 데이터 제공

| 필드 | 설명 | 필수 |
|-------|-------------|----------|
| 키 이름 | 이 키의 이름 | 예 |
| 개인 키 | 키 내용 붙여넣기 또는 "파일 선택" 클릭 | 예 |
| 설명 | 추가 메모 | 아니오 |

### 3단계: 가져오기
**"가져오기"** 클릭

## 공개 키 보기

1. 목록에서 키 클릭
2. 오른쪽에 세부 정보 패널 표시
3. **"공개 키 보기"** 클릭
4. 서버의 `~/.ssh/authorized_keys`에 추가할 공개 키 복사

## 키 내보내기

1. 목록에서 키 찾기
2. **내보내기** 버튼 클릭 (또는 마우스 오른쪽 클릭 → 내보내기)
3. 선택:
   - **공개 키만 내보내기** - 더 안전, 공유용
   - **두 키 모두 내보내기** - 개인 + 공개 (안전하게 보관!)
4. 저장 위치 선택
5. 키가 파일로 저장됨

## 키 삭제

1. 목록에서 키 찾기
2. **삭제** 버튼 (휴지통 아이콘) 클릭
3. 삭제 확인

> ⚠️ **경고:** 삭제된 키는 복구할 수 없습니다. 필요한 경우 백업이 있는지 확인하세요.

## 서버 연결에 키 사용

1. 서버 추가/편집 시
2. 인증을 **"SSH 키"**로 설정
3. **"키체인에서 SSH 키"** 드롭다운에서
4. 가져온 키 선택
5. 키에 암호구가 있으면 암호구 필드에 입력

## 서버에 공개 키 추가

비밀번호 없는 로그인을 활성화하려면 서버에 공개 키를 추가하세요:

### 방법 1: 수동
```bash
# 로컬 컴퓨터에서 공개 키 복사
# 그런 다음 서버에서:
echo "your-public-key-here" >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys
```

### 방법 2: ssh-copy-id 사용 (가능한 경우)
```bash
ssh-copy-id -i ~/.ssh/your_key.pub user@server
```

## 모범 사례

1. 가능하면 **ED25519 사용**
2. 추가 보안을 위해 **암호구 추가**
3. 다른 목적에 **다른 키 사용** (업무, 개인)
4. **키를 안전하게 백업**
5. **개인 키 절대 공유 금지** - 공개 키만 공유
6. 보안을 위해 **주기적으로 키 교체**
