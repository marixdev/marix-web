# RDP 원격 데스크톱

RDP (원격 데스크톱 프로토콜)를 사용하여 Windows 서버 및 컴퓨터에 연결합니다.

## RDP 서버 추가

1. **"호스트 추가"** 클릭
2. **프로토콜** 선택: RDP
3. 입력:
   - **호스트/IP**: Windows 서버 주소
   - **포트**: 3389 (기본값)
   - **사용자명**: Windows 사용자명
   - **비밀번호**: Windows 비밀번호
   - **도메인**: (선택 사항) Windows 도메인 이름
4. **생성** 클릭

## .rdp 파일에서 가져오기

이미 RDP 구성 파일이 있나요?

1. 서버 추가 대화 상자에서 **".rdp 파일 가져오기"** 클릭
2. `.rdp` 파일 선택
3. 파일에서 필드가 자동으로 채워짐
4. 필요한 경우 설정 조정
5. **생성** 클릭

## RDP에 연결

1. 호스트 목록에서 RDP 서버를 더블 클릭
2. 애플리케이션 내에서 원격 데스크톱 창이 열림
3. Windows 데스크톱과 정상적으로 상호 작용 가능

## RDP 컨트롤

| 입력 | 동작 |
|-------|----------|
| **마우스** | 클릭, 드래그, 스크롤 정상 작동 |
| **키보드** | 모든 키가 원격으로 전달됨 |
| **클립보드** | 로컬과 원격 간 복사/붙여넣기 |

## 연결 끊기

- 세션 탭에서 × 클릭
- 또는 애플리케이션 닫기
- 세션 상태는 원격 서버에 유지됨

## Linux 요구 사항

Linux에서 RDP 지원에는 FreeRDP가 필요합니다. Marix는 FreeRDP가 설치되어 있는지 **자동으로 감지**합니다.

### 자동 설치

FreeRDP가 발견되지 않으면:
1. 연결 시도 시 알림이 나타남
2. **"설치"** 버튼 클릭
3. Marix가 FreeRDP를 자동으로 설치
4. 설치 완료 후 재연결

### 수동 설치

수동으로 설치하려면:

**Ubuntu/Debian**
```bash
sudo apt install freerdp3-x11 xdotool
```

**Fedora**
```bash
sudo dnf install freerdp xdotool
```

**Arch Linux**
```bash
sudo pacman -S freerdp xdotool
```

## 디스플레이 설정

RDP 세션은 창 크기에 맞게 조정됩니다. 최상의 경험을 위해:
- 전체 데스크톱 경험을 위해 창 최대화
- 원격 해상도 조정을 위해 창 크기 조정
- 창 컨트롤을 통해 전체 화면 모드 사용 가능

## 성능 팁

- 느린 연결의 경우 **색상 깊이 감소**
- Windows 측에서 **시각 효과 비활성화**
- 최상의 성능을 위해 **유선 연결** 사용
- 원격 컴퓨터에서 불필요한 애플리케이션 닫기

## 문제 해결

### 연결 거부됨
- Windows에서 원격 데스크톱이 활성화되어 있는지 확인
- Windows 방화벽이 RDP를 허용하는지 확인
- 올바른 포트 확인 (기본값 3389)

### 검은 화면
- 잠시 기다리기 - 초기 연결에 시간이 걸릴 수 있음
- 재연결 시도
- 원격 컴퓨터가 절전 모드가 아닌지 확인

### 느린 성능
- 화면 해상도 낮추기
- 색상 깊이 줄이기
- 네트워크 속도 확인
- 대역폭을 많이 사용하는 애플리케이션 닫기

### 인증 실패
- 사용자명 형식 확인:
  - 로컬: `username`
  - 도메인: `DOMAIN\username` 또는 `username@domain.com`
- 비밀번호가 올바른지 확인
- 계정에 원격 데스크톱 접근 권한이 있는지 확인
