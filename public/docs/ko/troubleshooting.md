# 문제 해결

Marix의 일반적인 문제와 해결 방법입니다.

## 연결 문제

### "연결 거부됨"

**원인:**
- 서버가 실행 중이지 않음
- 잘못된 IP/호스트명
- 잘못된 포트
- 방화벽이 연결을 차단함

**해결 방법:**
1. 호스트/IP가 올바른지 확인
2. 포트가 올바르고 열려 있는지 확인
3. 서버가 실행 중인지 확인
4. 방화벽이 연결을 허용하는지 확인
5. 서버에 ping 시도

### "연결 시간 초과"

**원인:**
- 서버에 접근할 수 없음
- 네트워크 문제
- 방화벽이 차단함

**해결 방법:**
1. 인터넷 연결 확인
2. 서버가 온라인인지 확인
3. 다른 네트워크에서 시도
4. VPN이 필요한지 확인

### "인증 실패"

**원인:**
- 잘못된 사용자명
- 잘못된 비밀번호
- 잘못된 SSH 키
- 키 암호문구가 잘못됨

**해결 방법:**
1. 사용자명이 올바른지 확인
2. 비밀번호 확인 (대소문자 구분)
3. 올바른 SSH 키가 선택되었는지 확인
4. 올바른 키 암호문구 입력
5. 사용자가 서버에서 SSH 액세스 권한이 있는지 확인

### "호스트 키 검증 실패"

**원인:**
- 서버가 재설치됨
- 서버의 SSH 키가 변경됨
- 잠재적 보안 문제

**해결 방법:**
1. 서버 관리자에게 키가 변경되었는지 확인
2. 알려진 호스트 → 이전 항목 제거로 이동
3. 다시 연결하고 새 지문 수락
4. 의심스러우면 연결하기 전에 조사

### "권한 거부됨 (publickey)"

**원인:**
- SSH 키가 수락되지 않음
- 키가 authorized_keys에 없음
- 잘못된 키가 선택됨

**해결 방법:**
1. 공개 키가 서버의 `~/.ssh/authorized_keys`에 있는지 확인
2. 키 권한 확인: `chmod 600 ~/.ssh/authorized_keys`
3. Marix에서 올바른 키가 선택되었는지 확인
4. 사용 가능한 경우 비밀번호 인증 시도

## SFTP 문제

### 업로드 시 "권한 거부됨"

**원인:**
- 원격에 쓰기 권한 없음
- 잘못된 디렉토리
- 디스크 가득 참

**해결 방법:**
1. 원격 폴더에 쓰기 권한 확인
2. 먼저 홈 디렉토리에 업로드 시도
3. 서버의 디스크 공간 확인
4. 필요한 경우 `sudo` 사용 (터미널을 통해)

### 파일 전송 느림

**원인:**
- 네트워크 속도
- 큰 파일 크기
- 서버 부하

**해결 방법:**
1. 네트워크 속도 확인
2. 큰 파일은 시간이 더 걸립니다 - 인내심을 가지세요
3. 피크 시간이 아닐 때 시도
4. 전송 전에 파일 압축

### "해당 파일 또는 디렉토리 없음"

**원인:**
- 경로가 존재하지 않음
- 심볼릭 링크가 깨짐
- 디렉토리를 나열할 권한

**해결 방법:**
1. 경로가 존재하는지 확인
2. 위치로 수동으로 이동
3. 상위 디렉토리 권한 확인

## 터미널 문제

### 터미널에 깨진 텍스트 표시

**원인:**
- 인코딩 불일치
- 터미널 유형 비호환

**해결 방법:**
1. 다른 터미널 테마 시도
2. 서버의 로캘 설정 확인
3. 서버에 다시 연결

### 복사/붙여넣기가 작동하지 않음

**해결 방법:**
1. `Ctrl+Shift+C` 및 `Ctrl+Shift+V` 사용
2. **텍스트 복사**: 
   - 마우스로 텍스트 선택 → 복사
   - 또는 `Ctrl+C` 사용
3. **텍스트 붙여넣기**: 
   - 우클릭 → 붙여넣기
   - 또는 `Ctrl+V` 사용
4. 텍스트가 실제로 선택되었는지 확인

### 터미널이 응답하지 않음

**해결 방법:**
1. 잠시 기다리세요 - 명령이 처리 중일 수 있습니다
2. `Ctrl+C`를 눌러 실행 중인 명령 취소
3. 재연결 버튼 클릭
4. 탭을 닫았다가 다시 열기

## RDP 문제

### 연결 후 검은 화면

**원인:**
- 초기 연결 지연
- 원격 컴퓨터가 절전 모드
- 그래픽 문제

**해결 방법:**
1. 연결을 위해 잠시 기다리세요
2. 재연결 시도
3. 먼저 원격 컴퓨터를 깨우세요
4. 원격 컴퓨터가 잠겨 있지 않은지 확인

### RDP 성능 느림

**해결 방법:**
1. 색상 깊이 설정 줄이기
2. Windows에서 시각 효과 비활성화
3. 유선 연결 사용
4. 대역폭을 많이 사용하는 애플리케이션 닫기
5. 원격 해상도 낮추기

## 애플리케이션 문제

### Marix가 시작되지 않음

**해결 방법:**
1. 컴퓨터 재시작
2. Marix 재설치
3. 충돌하는 소프트웨어 확인
4. 관리자 권한으로 실행 (Windows)

### 설정이 저장되지 않음

**원인:**
- 쓰기 권한 없음
- 설정 파일 손상됨

**해결 방법:**
1. 관리자 권한으로 Marix 실행
2. 설정 재설정 (설정 → 재설정)
3. 설정 폴더 삭제 후 재시작

### 높은 CPU/메모리 사용량

**해결 방법:**
1. 사용하지 않는 연결 탭 닫기
2. 열려 있는 SFTP 패널 수 줄이기
3. Marix 재시작
4. 최신 버전으로 업데이트

## 백업 문제

### 복원 시 "잘못된 비밀번호"

**해결 방법:**
1. 정확한 비밀번호를 사용하고 있는지 확인
2. 비밀번호는 대소문자를 구분합니다
3. 앞뒤 공백 확인
4. 다른 제공자의 백업 시도

### 백업 업로드 실패

**해결 방법:**
1. 인터넷 연결 확인
2. OAuth 연결이 유효한지 확인
3. 백업 제공자에 다시 연결
4. 저장소 할당량이 초과되지 않았는지 확인

## 도움 받기

여기에 문제가 나열되지 않은 경우:

1. **업데이트 확인** - 최신 버전이 문제를 해결할 수 있습니다
2. GitHub에서 **이슈 검색**
3. [GitHub Issues](https://github.com/marixdev/marix/issues)에서 **버그 보고**
4. 웹사이트를 통해 **지원 문의**

문제 보고 시 포함할 내용:
- Marix 버전
- 운영 체제
- 재현 단계
- 오류 메시지 (있는 경우)
- 스크린샷 (해당하는 경우)
