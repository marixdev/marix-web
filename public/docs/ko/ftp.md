# FTP/FTPS 파일 전송

Marix는 SSH를 지원하지 않는 파일 서버에 연결하기 위한 FTP 및 FTPS 프로토콜을 지원합니다.

## 인터페이스 레이아웃

![FTP Interface](/ftp.png)

## FTP 서버 추가

1. **"호스트 추가"** 클릭
2. **프로토콜** 선택: FTP 또는 FTPS
3. 입력:
   - **호스트/IP**: FTP 서버 주소
   - **포트**: 21 (FTP) 또는 990 (FTPS)
   - **사용자명**: FTP 사용자명
   - **비밀번호**: FTP 비밀번호
4. **생성** 클릭

## FTP vs FTPS

| 기능 | FTP | FTPS |
|---------|-----|------|
| 암호화 | 없음 | TLS/SSL |
| 기본 포트 | 21 | 990 |
| 보안 | 낮음 | 높음 |
| 사용 사례 | 레거시 시스템 | 안전한 전송 |

> **권장**: 암호화된 파일 전송을 위해 가능한 경우 FTPS를 사용하세요.

## FTP에 연결

1. 호스트 목록에서 FTP 서버를 더블 클릭
2. 연결이 직접 **파일 브라우저 모드**로 열림
3. 터미널 사용 불가 (FTP는 파일 전송 전용)

## 파일 작업

모든 파일 작업은 SFTP와 동일하게 작동합니다:

### 파일 전송
- **드래그 앤 드롭**으로 로컬과 원격 간 전송
- **마우스 오른쪽 클릭** → 다운로드/업로드

### 파일 관리
- **폴더 생성**: 마우스 오른쪽 클릭 → 새 폴더
- **파일 이름 변경**: 마우스 오른쪽 클릭 → 이름 변경
- **파일 삭제**: 마우스 오른쪽 클릭 → 삭제

### 탐색
- 도구 모음 버튼 사용: 뒤로, 앞으로, 위로, 홈
- 폴더를 더블 클릭하여 들어가기
- 검색을 사용하여 파일 필터링

## 제한 사항

FTP는 다음을 지원하지 않습니다:
- **chmod** (권한 변경)
- **심볼릭 링크**
- **파일 편집** (다운로드, 편집, 다시 업로드 필요)

## 수동 vs 능동 모드

Marix는 기본적으로 **수동 모드**를 사용하며, 다음과 더 잘 작동합니다:
- NAT 라우터
- 방화벽
- 대부분의 현대 네트워크 구성

## 문제 해결

### 연결 시간 초과
- 서버가 귀하의 IP를 허용하는지 확인
- 양쪽 방화벽 규칙 확인
- 구성된 경우 다른 포트 시도

### 인증 실패
- 사용자명과 비밀번호 확인
- 서버에서 FTP 사용자가 활성화되어 있는지 확인
- 일부 서버는 특정 로그인 형식 필요

### 전송 실패
- 사용 가능한 디스크 공간 확인
- 서버에서 파일 권한 확인
- 대용량 파일은 시간 초과될 수 있음 - 서버 제한 확인
