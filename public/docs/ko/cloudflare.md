# Cloudflare DNS 관리자

Marix에서 직접 Cloudflare DNS 레코드를 관리하세요.

## Cloudflare DNS 관리자 인터페이스

![Cloudflare DNS 관리자 인터페이스](/cf.png)

## 설정

### 1단계: API 토큰 받기

1. [Cloudflare 대시보드](https://dash.cloudflare.com)로 이동
2. 프로필 → API 토큰으로 이동
3. "토큰 만들기" 클릭
4. "Edit zone DNS" 템플릿 사용 또는 다음 권한으로 사용자 정의 생성:
   - Zone.DNS: 편집
   - Zone.Zone: 읽기
5. 토큰 복사

### 2단계: Marix에서 설정

1. 사이드바에서 **설정**으로 이동
2. **"Cloudflare API 토큰"** 섹션 찾기
3. **"토큰 추가"** 클릭
4. 토큰 붙여넣기
5. **"저장"** 클릭

## Cloudflare DNS 사용

1. 사이드바에서 **"Cloudflare"** 클릭
2. 드롭다운에서 도메인이 로드됩니다
3. 도메인 선택
4. 해당 도메인의 DNS 레코드가 표시됩니다

## DNS 레코드 보기

테이블에 표시되는 항목:

| 열 | 설명 |
|--------|-------------|
| 유형 | 레코드 유형 (A, AAAA, CNAME, MX, TXT 등) |
| 이름 | 레코드 이름/서브도메인 |
| 내용 | 레코드 값 (IP, 호스트명, 텍스트) |
| TTL | Time-to-live (자동 또는 초) |
| 프록시 | Cloudflare를 통한 프록시 또는 DNS만 |

## DNS 레코드 추가

### 1단계: 대화상자 열기
**"레코드 추가"** 버튼 클릭

### 2단계: 상세 정보 입력

| 필드 | 설명 |
|-------|-------------|
| 유형 | 선택: A, AAAA, CNAME, MX, TXT, SRV 등 |
| 이름 | 서브도메인 (예: "www") 또는 루트의 경우 "@" |
| 내용 | 값 (A의 경우 IP, CNAME의 경우 호스트명 등) |
| TTL | Time-to-live: 자동 또는 사용자 정의 초 |
| 프록시 | 토글: 프록시됨 (주황색 구름) 또는 DNS만 (회색) |
| 코멘트 | 선택적 메모 |

### 3단계: 생성
**"생성"** 클릭

## 일반 레코드 유형

### A 레코드
도메인을 IPv4 주소로 연결합니다.

| 필드 | 예시 |
|-------|---------|
| 유형 | A |
| 이름 | @ (또는 www) |
| 내용 | 192.168.1.100 |

### AAAA 레코드
도메인을 IPv6 주소로 연결합니다.

| 필드 | 예시 |
|-------|---------|
| 유형 | AAAA |
| 이름 | @ |
| 내용 | 2001:db8::1 |

### CNAME 레코드
다른 도메인에 대한 별칭을 생성합니다.

| 필드 | 예시 |
|-------|---------|
| 유형 | CNAME |
| 이름 | www |
| 내용 | example.com |

### MX 레코드
메일 서버를 지정합니다.

| 필드 | 예시 |
|-------|---------|
| 유형 | MX |
| 이름 | @ |
| 내용 | mail.example.com |
| 우선순위 | 10 |

### TXT 레코드
텍스트 데이터를 저장합니다 (SPF, DKIM, 검증).

| 필드 | 예시 |
|-------|---------|
| 유형 | TXT |
| 이름 | @ |
| 내용 | v=spf1 include:_spf.google.com ~all |

## DNS 레코드 편집

1. 테이블에서 레코드 찾기
2. **편집** 버튼 (연필 아이콘) 클릭
3. 필드 수정
4. **"저장"** 클릭

## DNS 레코드 삭제

1. 레코드 찾기
2. **삭제** 버튼 (휴지통 아이콘) 클릭
3. 삭제 확인

## 프록시 상태

| 아이콘 | 상태 | 효과 |
|------|--------|--------|
| 🟠 주황색 구름 | 프록시됨 | 트래픽이 Cloudflare CDN을 통해 전달됨 |
| ⚪ 회색 구름 | DNS만 | 원본에 직접 연결 |

### 프록시 사용 시기

**프록시됨 (주황색):**
- 웹 서버 (HTTP/HTTPS)
- DDoS 보호가 필요할 때
- CDN 캐싱이 필요할 때

**DNS만 (회색):**
- SSH, FTP, 메일 서버
- HTTP가 아닌 서비스
- 직접 IP 연결이 필요할 때

## 레코드 새로고침

도메인 드롭다운 옆의 **새로고침** 버튼을 클릭하여 레코드를 다시 로드합니다.

## 팁

- **TTL 자동**은 대부분의 경우 권장됩니다
- DNS 조회 도구를 사용하여 **변경 사항 테스트**
- 주요 변경 전에 **레코드 백업**
- **전파**는 전 세계적으로 최대 48시간이 소요될 수 있습니다
