# 명령어 스니펫

스니펫은 SSH 터미널에서 자주 사용하는 명령어를 빠르게 실행할 수 있도록 도와주는 재사용 가능한 명령어 템플릿입니다.

## 명령어 스니펫 인터페이스

![명령어 스니펫 인터페이스](/snippets.png)

---

## 개요

스니펫 기능은 **명령어 템플릿**과 선택적 **키보드 단축키**를 결합합니다:

- 빠른 액세스를 위해 **자주 사용하는 명령어 저장**
- 즉시 실행을 위해 **단축키 할당** (Ctrl+Shift+키 또는 Cmd+Shift+키)
- **카테고리별 정리** (시스템, Docker, Git, 네트워크 등)
- **범위 기반 표시** - 전역, 호스트별 또는 그룹별

---

## 스니펫 접근

### 스니펫 패널

SSH 터미널에 연결하면 오른쪽에 **스니펫 패널**이 나타납니다:

- **검색** - 이름, 명령어 또는 태그로 스니펫 찾기
- **카테고리별 필터** - 카테고리 아이콘을 클릭하여 필터링
- **클릭하여 삽입** - 스니펫을 클릭하여 명령어 삽입
- **접기/펼치기** - 패널 표시 토글

### 스니펫 관리자

사이드바 메뉴에서 접근하여:

- 범위별로 정리된 모든 스니펫 보기
- 스니펫 추가, 편집 또는 삭제
- 단축키 할당 관리

---

## 스니펫 만들기

1. 사이드바에서 **스니펫 관리자** 열기
2. **스니펫 추가** 버튼 클릭
3. 양식 작성:

| 필드 | 설명 |
|-------|-------------|
| 이름 | 짧은 설명 제목 |
| 명령어 | 셸 명령어 |
| 카테고리 | 미리 정의된 것 또는 "사용자 정의" 선택 |
| 범위 | 전역, 호스트 또는 그룹 |
| 단축키 | 선택적 단일 문자 |
| 설명 | 명령어가 하는 일 설명 |
| 태그 | 검색을 위한 키워드 |

### 스니펫 예시

| 이름 | 명령어 | 카테고리 | 단축키 |
|------|---------|----------|--------|
| 디스크 사용량 | `df -h` | 시스템 | D |
| 모든 Docker 목록 | `docker ps -a` | Docker | P |
| Git 상태 | `git status` | Git | G |
| 포트 확인 | `netstat -tulpn` | 네트워크 | N |
| 로그 추적 | `tail -f /var/log/syslog` | 시스템 | L |

## 단축키 사용

### 키보드 단축키

| 플랫폼 | 단축키 형식 |
|----------|-----------------|
| **Windows/Linux** | `Ctrl + Shift + [키]` |
| **macOS** | `Cmd + Shift + [키]` |

### 예약된 키

일부 키는 예약되어 있어 사용할 수 없습니다:

| 키 | 예약 용도 |
|-----|--------------|
| A | 새 호스트 추가 |
| C | 복사 |
| L | LAN 전송 토글 |
| O | 터미널/SFTP 전환 |
| T | 로컬 터미널 |
| V | 붙여넣기 |

## 카테고리

| 아이콘 | 카테고리 | 용도 |
|------|----------|---------|
| 🖥️ | 시스템 | OS 명령어, 서비스, 로그 |
| 🐳 | Docker | 컨테이너 관리 |
| 📦 | Git | 버전 관리 |
| 🌐 | 네트워크 | 포트, 연결, DNS |
| 📁 | 파일 | 파일 작업 |
| 🗄️ | 데이터베이스 | 데이터베이스 명령어 |
| ✨ | 사용자 정의 | 기타 모든 것 |

## 범위 수준

### 전역 범위
**모든** SSH 연결에서 표시됩니다. 범용 명령어에 가장 적합합니다.

### 호스트 범위
**특정 서버**에서만 표시됩니다. 서버별 명령어에 가장 적합합니다.

### 그룹 범위
**특정 그룹**의 모든 서버에서 표시됩니다. 환경별 명령어에 가장 적합합니다.

---

## 팁 및 모범 사례

1. **변수 사용** - 변경되는 부분에 플레이스홀더 남기기: `tail -f [LOG_PATH]`
2. **명령어 연결** - 여러 명령어 결합: `cd /var/www && git pull && systemctl restart app`
3. **태그로 정리** - `prod`, `dev`, `urgent` 같은 태그 추가하여 쉽게 필터링
4. **스니펫 백업** - 암호화된 .marix 백업에 포함됨

---

## 데이터 저장

- 브라우저 localStorage에 로컬로 저장됨
- 암호화된 백업 (.marix 파일)에 포함됨
- Google Drive와 동기화됨 (활성화된 경우)
- 명시적인 백업 작업 없이는 클라우드 저장 없음

---

## 문제 해결

| 문제 | 해결 방법 |
|-------|----------|
| 단축키가 작동하지 않음 | 충돌 확인, 터미널에 포커스가 있는지 확인 |
| 스니펫이 나타나지 않음 | 범위가 현재 연결과 일치하는지 확인 |
| 명령어가 실행되지 않음 | 터미널 상태, 권한 문제 확인 |
