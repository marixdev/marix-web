# Передача файлов FTP/FTPS

Marix поддерживает протоколы FTP и FTPS для подключения к файловым серверам, не поддерживающим SSH.

## Макет интерфейса

![Интерфейс FTP](/ftp.png)

## Добавление FTP-сервера

1. Нажмите **«Добавить новый хост»**
2. Выберите **Протокол**: FTP или FTPS
3. Заполните:
   - **Хост/IP**: Адрес FTP-сервера
   - **Порт**: 21 (FTP) или 990 (FTPS)
   - **Имя пользователя**: Имя пользователя FTP
   - **Пароль**: Пароль FTP
4. Нажмите **Создать**

## FTP vs FTPS

| Функция | FTP | FTPS |
|---------|-----|------|
| Шифрование | Нет | TLS/SSL |
| Порт по умолчанию | 21 | 990 |
| Безопасность | Низкая | Высокая |
| Применение | Устаревшие системы | Безопасная передача |

> **Рекомендация**: Используйте FTPS когда возможно для зашифрованной передачи файлов.

## Подключение к FTP

1. Дважды щёлкните FTP-сервер в списке хостов
2. Подключение открывается напрямую в **режиме файлового браузера**
3. Терминал недоступен (FTP только для передачи файлов)

## Файловые операции

Все файловые операции работают так же, как в SFTP:

### Передача файлов
- **Перетащите** для передачи между локальным и удалённым
- **Правый клик** → Скачать/Загрузить

### Управление файлами
- **Создать папку**: Правый клик → Новая папка
- **Переименовать файлы**: Правый клик → Переименовать
- **Удалить файлы**: Правый клик → Удалить

### Навигация
- Используйте кнопки панели инструментов: Назад, Вперёд, Вверх, Домой
- Дважды щёлкните папку для входа
- Используйте поиск для фильтрации файлов

## Ограничения

FTP не поддерживает:
- **chmod** (изменение прав доступа)
- **Символические ссылки**
- **Редактирование файлов** (нужно скачать, отредактировать, загрузить обратно)

## Пассивный vs Активный режим

Marix использует **пассивный режим** по умолчанию, который лучше работает с:
- NAT-маршрутизаторами
- Файрволами
- Большинством современных сетевых конфигураций

## Устранение неполадок

### Тайм-аут подключения
- Проверьте, разрешает ли сервер ваш IP
- Проверьте правила файрвола с обеих сторон
- Попробуйте другие порты, если настроены

### Ошибка аутентификации
- Проверьте имя пользователя и пароль
- Проверьте, включён ли пользователь FTP на сервере
- Некоторые серверы требуют специфические форматы логина

### Ошибка передачи
- Проверьте доступное место на диске
- Проверьте права доступа к файлам на сервере
- Большие файлы могут вызвать тайм-аут — проверьте лимиты сервера
