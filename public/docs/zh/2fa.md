# 双因素认证（2FA/TOTP）

内置 TOTP 代码生成器，类似 Google Authenticator。存储您的 2FA 密钥并获取验证码，无需手机。

## 2FA 界面

![2FA Interface](/2fa.png)

## 打开 2FA

在侧边栏中点击 **"2FA"**。

## 添加新的 2FA 账户

### 步骤 1：打开对话框
点击 **"添加账户"** 按钮

### 步骤 2：输入详情

| 字段 | 描述 | 必填 |
|-------|-------------|----------|
| 账户名称 | 此账户的标签（例如"GitHub"、"AWS"） | 否（自动命名） |
| 密钥 | TOTP 密钥（来自二维码设置页面） | 是 |

### 步骤 3：保存
点击 **"添加"**

## 如何获取密钥

在网站上启用 2FA 时：

1. 查找 **"无法扫描二维码？"** 或 **"手动输入"**
2. 复制密钥（通常是大写字母和数字）
3. 粘贴到 Marix

示例密钥格式：
```
JBSWY3DPEHPK3PXP
```

## 使用 2FA 验证码

1. 在列表中找到您的账户
2. 6 位验证码显示在旁边
3. 倒计时器显示验证码刷新剩余秒数
4. 点击 **复制** 按钮将验证码复制到剪贴板

### 计时器
- 验证码每 **30 秒** 刷新一次
- 圆形进度指示器显示剩余时间
- 如果当前验证码即将过期，请等待新验证码

## 编辑账户名称

1. 点击账户旁边的 **编辑** 按钮
2. 输入新名称
3. 按 Enter 或点击保存

## 删除 2FA 账户

1. 点击 **删除** 按钮（垃圾桶图标）
2. 确认删除

> ⚠️ **警告：** 删除会移除密钥。在删除之前，请确保您有其他方式访问关联的账户。

## 常用服务

以下是一些支持 TOTP 的常用服务：

| 服务 | 2FA 设置位置 |
|---------|----------------------|
| GitHub | 设置 → 密码和认证 |
| AWS | IAM → 安全凭证 |
| Google | Google 账户 → 安全性 |
| Microsoft | 安全设置 → 两步验证 |
| Cloudflare | 个人资料 → 认证 |
| DigitalOcean | 账户 → 安全 |
| Linode | 我的个人资料 → 密码和认证 |

## 最佳实践

1. **备份您的密钥** - 安全存储密钥
2. **保留恢复码** - 保存服务提供的备份码
3. **使用描述性名称** - 便于快速识别账户
4. **添加后立即测试验证码** - 验证是否正常工作
5. **时间同步** - 确保您的计算机时钟准确

## 故障排除

### 验证码不工作

1. **检查时间同步** - 您的计算机时钟必须准确
2. **验证密钥** - 如果复制错误，请重新输入
3. **等待下一个验证码** - 当前验证码可能即将过期

### 丢失访问权限

如果您无法访问 Marix：
1. 使用服务提供的 **恢复码**
2. 联系服务支持进行账户恢复
