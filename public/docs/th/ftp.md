# โอนไฟล์ FTP/FTPS

Marix รองรับโปรโตคอล FTP และ FTPS สำหรับเชื่อมต่อกับเซิร์ฟเวอร์ไฟล์ที่ไม่รองรับ SSH

## เลย์เอาต์หน้าจอ

![หน้าจอ FTP](/ftp.png)

## เพิ่มเซิร์ฟเวอร์ FTP

1. คลิก **"เพิ่มโฮสต์ใหม่"**
2. เลือก **โปรโตคอล**: FTP หรือ FTPS
3. กรอก:
   - **Host/IP**: ที่อยู่เซิร์ฟเวอร์ FTP
   - **Port**: 21 (FTP) หรือ 990 (FTPS)
   - **Username**: ชื่อผู้ใช้ FTP
   - **Password**: รหัสผ่าน FTP
4. คลิก **สร้าง**

## FTP vs FTPS

| ฟีเจอร์ | FTP | FTPS |
|---------|-----|------|
| การเข้ารหัส | ไม่มี | TLS/SSL |
| พอร์ตเริ่มต้น | 21 | 990 |
| ความปลอดภัย | ต่ำ | สูง |
| กรณีใช้งาน | ระบบเก่า | การโอนที่ปลอดภัย |

> **แนะนำ**: ใช้ FTPS เมื่อมีสำหรับการโอนไฟล์ที่เข้ารหัส

## เชื่อมต่อกับ FTP

1. ดับเบิลคลิกเซิร์ฟเวอร์ FTP ในรายการ Hosts
2. เปิดการเชื่อมต่อโดยตรงใน **โหมดเรียกดูไฟล์**
3. ไม่มีเทอร์มินัล (FTP ใช้สำหรับโอนไฟล์เท่านั้น)

## การดำเนินการไฟล์

การดำเนินการไฟล์ทั้งหมดทำงานเหมือน SFTP:

### โอนไฟล์
- **ลากและวาง** เพื่อโอนระหว่างเครื่องและรีโมท
- **คลิกขวา** → ดาวน์โหลด/อัปโหลด

### จัดการไฟล์
- **สร้างโฟลเดอร์**: คลิกขวา → โฟลเดอร์ใหม่
- **เปลี่ยนชื่อไฟล์**: คลิกขวา → เปลี่ยนชื่อ
- **ลบไฟล์**: คลิกขวา → ลบ

### การนำทาง
- ใช้ปุ่มแถบเครื่องมือ: ย้อนกลับ, ไปข้างหน้า, ขึ้น, หน้าแรก
- ดับเบิลคลิกโฟลเดอร์เพื่อเข้า
- ใช้ค้นหาเพื่อกรองไฟล์

## ข้อจำกัด

FTP ไม่รองรับ:
- **chmod** (เปลี่ยนสิทธิ์)
- **Symbolic links**
- **แก้ไขไฟล์** (ต้องดาวน์โหลด, แก้ไข, อัปโหลดใหม่)

## Passive vs Active Mode

Marix ใช้ **passive mode** โดยค่าเริ่มต้น ซึ่งทำงานได้ดีกับ:
- NAT routers
- Firewalls
- การกำหนดค่าเครือข่ายสมัยใหม่ส่วนใหญ่

## การแก้ไขปัญหา

### Connection Timeout
- ตรวจสอบว่าเซิร์ฟเวอร์อนุญาต IP ของคุณ
- ตรวจสอบกฎ firewall ทั้งสองฝั่ง
- ลองใช้พอร์ตอื่นถ้ามีการกำหนดค่า

### Authentication Failed
- ตรวจสอบชื่อผู้ใช้และรหัสผ่าน
- ตรวจสอบว่าผู้ใช้ FTP เปิดใช้งานบนเซิร์ฟเวอร์
- บางเซิร์ฟเวอร์ต้องการรูปแบบการเข้าสู่ระบบเฉพาะ
