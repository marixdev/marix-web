# เครื่องมือเครือข่าย

ชุดเครื่องมือสำหรับวินิจฉัยและตรวจสอบเครือข่าย

## หน้าจอเครื่องมือเครือข่าย

![หน้าจอเครื่องมือเครือข่าย](/networktools.png)

## เครื่องมือที่รองรับ

### Ping
ตรวจสอบการเชื่อมต่อกับโฮสต์และวัดเวลาตอบสนอง

**วิธีใช้:**
1. ไปที่ **เครื่องมือเครือข่าย** → **Ping**
2. ใส่ที่อยู่ IP หรือชื่อโฮสต์
3. คลิก **"Ping"**
4. ดูผลลัพธ์:
   - เวลาตอบสนอง (ms)
   - Packet loss
   - สถานะการเชื่อมต่อ

### Traceroute
ติดตามเส้นทางแพ็กเก็ตไปยังปลายทาง

**วิธีใช้:**
1. ไปที่ **เครื่องมือเครือข่าย** → **Traceroute**
2. ใส่ที่อยู่ปลายทาง
3. คลิก **"Trace"**
4. ดูผลลัพธ์:
   - แต่ละ hop พร้อม IP
   - เวลาตอบสนองแต่ละจุด
   - จำนวน hops ทั้งหมด

### DNS Lookup
ค้นหาข้อมูล DNS ของโดเมน

**วิธีใช้:**
1. ไปที่ **เครื่องมือเครือข่าย** → **DNS Lookup**
2. ใส่ชื่อโดเมน
3. เลือกประเภท record:
   - A (IPv4)
   - AAAA (IPv6)
   - MX (Mail)
   - TXT (Text)
   - NS (Name Server)
   - CNAME
4. คลิก **"Lookup"**

### Port Scanner
สแกนพอร์ตที่เปิดบนโฮสต์

**วิธีใช้:**
1. ไปที่ **เครื่องมือเครือข่าย** → **Port Scanner**
2. ใส่ที่อยู่ IP หรือชื่อโฮสต์
3. ระบุช่วงพอร์ต (เช่น 1-1000 หรือ 22,80,443)
4. คลิก **"Scan"**
5. ดูผลลัพธ์:
   - พอร์ตที่เปิด
   - บริการที่ตรวจพบ

### Whois
ค้นหาข้อมูลการจดทะเบียนโดเมนหรือ IP

**วิธีใช้:**
1. ไปที่ **เครื่องมือเครือข่าย** → **Whois**
2. ใส่ชื่อโดเมนหรือ IP
3. คลิก **"Lookup"**
4. ดูผลลัพธ์:
   - ผู้จดทะเบียน
   - วันที่จดทะเบียน/หมดอายุ
   - ข้อมูลติดต่อ

### Wake-on-LAN
ปลุกเครื่องคอมพิวเตอร์ผ่านเครือข่าย

**วิธีใช้:**
1. ไปที่ **เครื่องมือเครือข่าย** → **Wake-on-LAN**
2. ใส่ MAC address ของเครื่องเป้าหมาย
3. (ไม่บังคับ) ระบุ IP broadcast
4. คลิก **"Wake"**

> หมายเหตุ: เครื่องเป้าหมายต้องรองรับ WOL และเปิดใช้งานใน BIOS

## การตั้งค่าเครื่องมือ

| ตัวเลือก | คำอธิบาย |
|--------|-------------|
| Timeout | เวลารอสูงสุดก่อน timeout |
| จำนวน Ping | จำนวนครั้งที่ ping |
| ขนาด Packet | ขนาด packet สำหรับ ping |

## บันทึกผลลัพธ์

1. หลังจากรันเครื่องมือ
2. คลิก **"บันทึก"** หรือ **"คัดลอก"**
3. ผลลัพธ์จะถูกบันทึกเป็นไฟล์หรือคัดลอกไปยังคลิปบอร์ด

## แก้ปัญหา

### Ping ไม่สำเร็จ
- ตรวจสอบการเชื่อมต่ออินเทอร์เน็ต
- ยืนยันว่าโฮสต์ปลายทางออนไลน์
- ตรวจสอบ firewall ที่อาจบล็อก ICMP

### Port Scan ช้า
- ลดช่วงพอร์ตที่สแกน
- เพิ่ม timeout
- ตรวจสอบแบนด์วิดท์เครือข่าย

### DNS Lookup ล้มเหลว
- ตรวจสอบการสะกดโดเมน
- ลองใช้ DNS server อื่น
- ยืนยันการเชื่อมต่ออินเทอร์เน็ต
