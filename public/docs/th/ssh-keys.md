# ตัวจัดการ SSH Key

จัดการ SSH keys สำหรับการยืนยันตัวตนโดยไม่ต้องใช้รหัสผ่านไปยังเซิร์ฟเวอร์ของคุณ

## หน้าจอตัวจัดการ SSH Key

![หน้าจอตัวจัดการ SSH Key](/sshkey.png)

## เปิดตัวจัดการ SSH Key

คลิก **"SSH Key"** ในเมนูแถบด้านข้าง

## สร้าง SSH Key ใหม่

### ขั้นตอนที่ 1: เปิดตัวสร้าง
คลิกปุ่ม **"สร้าง Key"**

### ขั้นตอนที่ 2: กรอกรายละเอียด

| ฟิลด์ | คำอธิบาย | จำเป็น |
|-------|-------------|----------|
| ชื่อ Key | ชื่อเพื่อระบุ key นี้ (เช่น "work-laptop") | ใช่ |
| ประเภท Key | ed25519 (แนะนำ), RSA หรือ ECDSA | ใช่ |
| Passphrase | รหัสผ่านเพื่อเข้ารหัส key (ไม่บังคับแต่แนะนำ) | ไม่ |
| Comment | หมายเหตุเพิ่มเติม (เช่น ที่อยู่อีเมล) | ไม่ |

### ขั้นตอนที่ 3: สร้าง
คลิก **"สร้าง"**

### ขั้นตอนที่ 4: เสร็จสิ้น
Key ปรากฏในรายการของคุณพร้อม:
- ชื่อ
- ป้ายประเภท (ED25519, RSA, ECDSA)
- Fingerprint
- วันที่สร้าง

## เปรียบเทียบประเภท Key

| ประเภท | ความปลอดภัย | ความเข้ากันได้ | คำแนะนำ |
|------|----------|---------------|----------------|
| **ED25519** | ยอดเยี่ยม | เซิร์ฟเวอร์สมัยใหม่ | ✅ แนะนำ |
| **RSA (4096)** | ดีมาก | ใช้งานได้ทั่วไป | ดีสำหรับระบบเก่า |
| **ECDSA** | ดี | เซิร์ฟเวอร์ส่วนใหญ่ | ทางเลือกของ ED25519 |

## Import Key ที่มีอยู่

### ขั้นตอนที่ 1: เปิด Importer
คลิกปุ่ม **"Import Key"**

### ขั้นตอนที่ 2: ให้ข้อมูล Key

| ฟิลด์ | คำอธิบาย | จำเป็น |
|-------|-------------|----------|
| ชื่อ Key | ชื่อสำหรับ key นี้ | ใช่ |
| Private Key | วางเนื้อหา key หรือคลิก "เลือกไฟล์" | ใช่ |

### ขั้นตอนที่ 3: Import
คลิก **"Import"**

## ดู Public Key

1. คลิก key ในรายการ
2. แผงรายละเอียดแสดงทางขวา
3. คลิก **"แสดง Public Key"**
4. คัดลอก public key เพื่อเพิ่มไปยัง `~/.ssh/authorized_keys` ของเซิร์ฟเวอร์

## Export Key

1. ค้นหา key ในรายการ
2. คลิกปุ่ม **Export** (หรือคลิกขวา → Export)
3. เลือก:
   - **Export Public Key Only** - ปลอดภัยกว่า, สำหรับแชร์
   - **Export Both Keys** - Private + Public (เก็บให้ปลอดภัย!)
4. เลือกตำแหน่งบันทึก

## ลบ Key

1. ค้นหา key ในรายการ
2. คลิกปุ่ม **ลบ** (ไอคอนถังขยะ)
3. ยืนยันการลบ

> ⚠️ **คำเตือน:** Key ที่ลบไม่สามารถกู้คืนได้ ตรวจสอบให้แน่ใจว่าคุณมีการสำรองถ้าจำเป็น

## ใช้ Key สำหรับการเชื่อมต่อเซิร์ฟเวอร์

1. เมื่อเพิ่ม/แก้ไขเซิร์ฟเวอร์
2. ตั้ง Authentication เป็น **"SSH Key"**
3. ในเมนูดร็อปดาวน์ **"SSH Key from Keychain"**
4. เลือก key ที่ import ไว้
5. ถ้า key มี passphrase กรอกในฟิลด์ Passphrase

## เพิ่ม Public Key ไปยังเซิร์ฟเวอร์

เพื่อเปิดใช้งานการเข้าสู่ระบบโดยไม่ต้องใช้รหัสผ่าน เพิ่ม public key ไปยังเซิร์ฟเวอร์:

### วิธีที่ 1: ด้วยตนเอง
```bash
# บนเครื่องของคุณ, คัดลอก public key
# จากนั้นบนเซิร์ฟเวอร์:
echo "your-public-key-here" >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys
```

### วิธีที่ 2: ใช้ ssh-copy-id (ถ้ามี)
```bash
ssh-copy-id -i ~/.ssh/your_key.pub user@server
```

## แนวปฏิบัติที่ดี

1. **ใช้ ED25519** สำหรับ key ใหม่เมื่อเป็นไปได้
2. **เพิ่ม passphrase** เพื่อความปลอดภัยเพิ่มเติม
3. **ใช้ key ต่างกัน** สำหรับวัตถุประสงค์ต่างกัน (งาน, ส่วนตัว)
4. **สำรอง key ของคุณ** อย่างปลอดภัย
5. **อย่าแชร์ private key** - แชร์เฉพาะ public key
6. **หมุนเวียน key เป็นระยะ** เพื่อความปลอดภัย
