# การแก้ปัญหา

คำแนะนำในการแก้ปัญหาที่พบบ่อย

## ปัญหาการเชื่อมต่อ

### ไม่สามารถเชื่อมต่อกับเซิร์ฟเวอร์

**อาการ:** ข้อความ "Connection refused" หรือ "Connection timed out"

**วิธีแก้:**
1. ตรวจสอบที่อยู่ IP/hostname ถูกต้อง
2. ยืนยันว่าเซิร์ฟเวอร์ออนไลน์
3. ตรวจสอบพอร์ต (ค่าเริ่มต้น SSH คือ 22)
4. ตรวจสอบ firewall ทั้งฝั่งเซิร์ฟเวอร์และเครื่องคุณ
5. ลอง ping เซิร์ฟเวอร์

```bash
ping your-server-ip
```

### การยืนยันตัวตนล้มเหลว

**อาการ:** "Authentication failed" หรือ "Permission denied"

**วิธีแก้:**
1. ตรวจสอบชื่อผู้ใช้ถูกต้อง
2. ตรวจสอบรหัสผ่าน (ระวังตัวพิมพ์ใหญ่-เล็ก)
3. ถ้าใช้ SSH key:
   - ยืนยันว่า public key อยู่บน `~/.ssh/authorized_keys` ของเซิร์ฟเวอร์
   - ตรวจสอบ passphrase ของ key
   - ยืนยันสิทธิ์ไฟล์: `chmod 600 ~/.ssh/authorized_keys`

### การเชื่อมต่อหลุดบ่อย

**อาการ:** การเชื่อมต่อถูกตัดโดยไม่คาดคิด

**วิธีแก้:**
1. เปิดใช้ keep-alive ในการตั้งค่า
2. ตรวจสอบการเชื่อมต่ออินเทอร์เน็ต
3. เพิ่มค่า timeout
4. ตรวจสอบ server-side idle timeout

## ปัญหา Terminal

### ตัวอักษรแสดงผิดพลาด

**อาการ:** ตัวอักษรเป็นสี่เหลี่ยมหรือแสดงผิด

**วิธีแก้:**
1. ไปที่ **การตั้งค่า** → **Terminal**
2. เปลี่ยนฟอนต์เป็น monospace font ที่รองรับ Unicode
3. ตรวจสอบ encoding เป็น UTF-8

### สีไม่แสดง

**อาการ:** Terminal ไม่แสดงสีของคำสั่ง

**วิธีแก้:**
1. ยืนยันว่า terminal รองรับ 256 สี
2. ตรวจสอบตัวแปร TERM:
```bash
echo $TERM
```
3. ตั้งค่า TERM ถ้าจำเป็น:
```bash
export TERM=xterm-256color
```

### ขนาดหน้าจอไม่ถูกต้อง

**อาการ:** หน้าจอ terminal มีขนาดผิดพลาด

**วิธีแก้:**
1. ปรับขนาดหน้าต่าง
2. รันคำสั่ง:
```bash
resize
```
หรือ
```bash
stty rows 24 cols 80
```

## ปัญหา SFTP

### ไม่สามารถอัปโหลด/ดาวน์โหลดไฟล์

**อาการ:** การโอนไฟล์ล้มเหลว

**วิธีแก้:**
1. ตรวจสอบสิทธิ์การเขียนโฟลเดอร์ปลายทาง
2. ยืนยันพื้นที่ดิสก์เพียงพอ
3. ตรวจสอบขนาดไฟล์ (อาจมีขีดจำกัด)
4. ลองโอนไฟล์ขนาดเล็กเพื่อทดสอบ

### ไม่สามารถลบไฟล์

**อาการ:** "Permission denied" เมื่อลบ

**วิธีแก้:**
1. ตรวจสอบสิทธิ์ไฟล์:
```bash
ls -la filename
```
2. เปลี่ยนสิทธิ์ถ้าจำเป็น:
```bash
chmod 644 filename
```

## ปัญหา SSH Key

### Key ไม่ทำงาน

**อาการ:** เซิร์ฟเวอร์ไม่รับ SSH key

**วิธีแก้:**
1. ตรวจสอบว่า public key อยู่บนเซิร์ฟเวอร์
2. ยืนยันสิทธิ์:
```bash
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys
```
3. ตรวจสอบ SELinux/AppArmor ที่อาจบล็อก

### Passphrase ลืม

**วิธีแก้:**
- ไม่สามารถกู้คืน passphrase ได้
- สร้าง key ใหม่และเพิ่มไปยังเซิร์ฟเวอร์

## ปัญหาประสิทธิภาพ

### แอปพลิเคชันช้า

**วิธีแก้:**
1. ปิดแท็บที่ไม่ใช้
2. ล้าง cache:
   - ไปที่ **การตั้งค่า** → **ขั้นสูง** → **ล้าง Cache**
3. รีสตาร์ทแอปพลิเคชัน
4. ตรวจสอบการใช้หน่วยความจำ

### การโอนไฟล์ช้า

**วิธีแก้:**
1. ตรวจสอบแบนด์วิดท์เครือข่าย
2. ลดจำนวนการโอนพร้อมกัน
3. ใช้การบีบอัดถ้ามี
4. ลองช่วงเวลาอื่น

## ปัญหาการสำรองข้อมูล

### Sync ล้มเหลว

**วิธีแก้:**
1. ตรวจสอบการเชื่อมต่อกับ provider (Google Drive, GitHub, ฯลฯ)
2. ยืนยันการ login
3. ตรวจสอบพื้นที่เก็บข้อมูล
4. ลอง sync ใหม่

### กู้คืนไม่สำเร็จ

**วิธีแก้:**
1. ยืนยันว่าไฟล์สำรองยังมีอยู่
2. ตรวจสอบ format ไฟล์
3. ลองกู้คืนทีละส่วน

## การติดต่อสนับสนุน

หากยังพบปัญหา:
1. รวบรวมข้อมูลเกี่ยวกับปัญหา
2. บันทึก logs ที่เกี่ยวข้อง
3. ติดต่อทีมสนับสนุน

## เคล็ดลับทั่วไป

1. **รีสตาร์ทก่อน** - หลายปัญหาแก้ได้ด้วยการรีสตาร์ท
2. **อัปเดตเวอร์ชัน** - ใช้เวอร์ชันล่าสุดเสมอ
3. **ตรวจสอบ logs** - logs มักบอกสาเหตุของปัญหา
4. **ทดสอบทีละขั้น** - แยกส่วนที่มีปัญหา
5. **สำรองข้อมูล** - ก่อนทำการเปลี่ยนแปลงสำคัญ
