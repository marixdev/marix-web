# โฮสต์ที่รู้จัก

โฮสต์ที่รู้จักเก็บลายนิ้วมือเซิร์ฟเวอร์เพื่อตรวจจับว่าตัวตนของเซิร์ฟเวอร์เปลี่ยนไป (ปัญหาความปลอดภัยที่อาจเกิดขึ้น)

## หน้าจอโฮสต์ที่รู้จัก

![หน้าจอโฮสต์ที่รู้จัก](/knowhosts.png)

## โฮสต์ที่รู้จักคืออะไร?

เมื่อคุณเชื่อมต่อกับเซิร์ฟเวอร์ SSH ครั้งแรก เซิร์ฟเวอร์จะแสดงลายนิ้วมือ Marix เก็บสิ่งนี้เพื่อ:
- ตรวจสอบตัวตนเซิร์ฟเวอร์ในการเชื่อมต่อในอนาคต
- ตรวจจับการโจมตีแบบ man-in-the-middle
- แจ้งเตือนคุณหาก key ของเซิร์ฟเวอร์เปลี่ยนโดยไม่คาดคิด

## เปิดโฮสต์ที่รู้จัก

คลิก **"โฮสต์ที่รู้จัก"** ในแถบด้านข้าง

## ดูโฮสต์ที่รู้จัก

รายการแสดง:

| คอลัมน์ | คำอธิบาย |
|--------|-------------|
| Host | hostname/IP ของเซิร์ฟเวอร์ |
| Port | พอร์ต SSH |
| Key Type | อัลกอริทึม SSH key (ED25519, RSA, ECDSA) |
| Fingerprint | ตัวระบุเซิร์ฟเวอร์ที่ไม่ซ้ำ |
| Added | เมื่อโฮสต์ถูกเชื่อถือ |

## ค้นหาโฮสต์ที่รู้จัก

1. ใช้ช่องค้นหาด้านบน
2. ค้นหาตาม hostname, IP หรือ fingerprint

## กรองตามประเภท Key

1. ใช้ตัวกรองแบบดร็อปดาวน์
2. เลือก: ทั้งหมด, ED25519, RSA หรือ ECDSA

## Import โฮสต์จากเซิร์ฟเวอร์

เพิ่มลายนิ้วมือของเซิร์ฟเวอร์ด้วยตนเองโดยไม่ต้องเชื่อมต่อ:

1. คลิกปุ่ม **"Import from Host"**
2. กรอก:
   - Hostname หรือ IP
   - Port (ค่าเริ่มต้น 22)
3. คลิก **"Import"**
4. ลายนิ้วมือของเซิร์ฟเวอร์จะถูกดึงและเก็บ

## ลบโฮสต์ที่รู้จัก

1. ค้นหาโฮสต์ในรายการ
2. คลิกปุ่ม **ลบ** (ไอคอนถังขยะ)
3. ยืนยันการลบ

### เมื่อใดควรลบ

- เซิร์ฟเวอร์ถูกติดตั้งใหม่และมี key ใหม่
- IP/hostname ของเซิร์ฟเวอร์เปลี่ยน
- คุณไม่เชื่อมต่อกับเซิร์ฟเวอร์นี้อีกต่อไป

## คำเตือนความปลอดภัย

### "Host key has changed"
คำเตือนนี้ปรากฏเมื่อลายนิ้วมือของเซิร์ฟเวอร์แตกต่างจากที่เก็บไว้

**สาเหตุที่เป็นไปได้:**
- เซิร์ฟเวอร์ถูกติดตั้งใหม่
- SSH key ของเซิร์ฟเวอร์ถูกสร้างใหม่
- ⚠️ **การโจมตีแบบ man-in-the-middle** (หายากแต่เป็นไปได้)

**ควรทำอย่างไร:**
1. ตรวจสอบกับผู้ดูแลเซิร์ฟเวอร์ว่า key เปลี่ยนอย่างถูกต้อง
2. ถ้ายืนยันแล้ว ลบรายการเก่าจากโฮสต์ที่รู้จัก
3. เชื่อมต่อใหม่และยอมรับลายนิ้วมือใหม่
4. ถ้าสงสัย อย่าเชื่อมต่อ

## แนวปฏิบัติที่ดี

1. **ตรวจสอบลายนิ้วมือ** เมื่อเชื่อมต่อกับเซิร์ฟเวอร์ใหม่
2. **อย่าเพิกเฉยคำเตือน** เกี่ยวกับ host key ที่เปลี่ยน
3. **ตรวจสอบเป็นประจำ** รายชื่อโฮสต์ที่รู้จัก
4. **ลบรายการเก่า** สำหรับเซิร์ฟเวอร์ที่คุณไม่ใช้แล้ว
